# WGTG22 Bot

Telegram-бот для интеграции с WG-Easy.
Развертывание производится с помощью PM2 (для Linux-серверов).

## Предварительные требования
- Node.js >= 20
- PM2 (`npm install -g pm2`)

## Конфигурация (.env)
В папке с проектом необходимо создать файл `.env` со следующим содержимым:

```env
TELEGRAM_BOT_TOKEN="ВАШ_ТОКЕН"
WG_EASY_URL="http://IP_АДРЕС:51821/"
WG_EASY_PASSWORD="ПАРОЛЬ_ОТ_WG_EASY"
ADMIN_IDS="ID1,ID2,ID3"
```
*(Этот файл добавлен в `.gitignore` и не будет загружаться на GitHub для вашей безопасности).*

## 🚀 Установка в одну команду (Deploy)

Если вы загрузили этот проект на GitHub, то для развертывания бота на абсолютно пустом сервере Linux достаточно ввести всего **одну команду**:

```bash
git clone https://github.com/NGPpQr0oWJ12/wgtg22.git && cd wgtg22 && bash deploy/setup.sh
```

**Что делает эта команда?**
1. Скачивает исходный код бота.
2. Автоматически устанавливает `Node.js 22` и `PM2` (если их нет на сервере).
3. Интерактивно спрашивает у вас `TELEGRAM_BOT_TOKEN`, URL от WG-Easy и другие ключи, после чего сама сохраняет их в защищенный файл `.env`.
4. Загружает необходимые библиотеки (`npm install`).
5. Запускает бота в правильном режиме через `pm2 start ecosystem.config.js` и заставляет его работать после перезагрузки сервера (`pm2 save`).

## Запуск для разработчиков
Бот настроен на предотвращение конфликтов (очистка webhooks) и принудительное подключение к API Telegram по протоколу IPv4 (через патч DNS), что предотвращает частые сбои AggregateError.

```bash
npm install
pm2 start ecosystem.config.js
```
